Tuto installation git et utilisation de gitlab

Dans ce tutoriel, vous apprendrez à installer Git sur votre machine, lier votre clé SSH à votre compte GitLab et quelques commandes de base.

Installation

Windows

Télécharger l'installateur ici, et suivre les instructions. Pensez à laisser activer (ou l'activer si cas échéant) l'option pour rajouter les commandes git directement dans l'invite de commande Windows.

Linux

Git est installé de base sur la plupart des distributions. Si cependant ce n'est pas le cas pour vous, les commandes d'installation sont disponibles ici (à choisir selon votre gestionnaire de paquets).

MacOS

Installez homebrew, puis lancez la commande brew install git.

Générer et lier la clé ssh

Pour générer la clé ssh, entrez simplement ssh-keygen dans votre terminal. On devrait vous demander un mot de passe pour sécuriser la clé. Vous pouvez en choisir un (qui vous sera demandé à chaque opération git). Sinon, vous pouvez laisser le champ vide. (Il faut entrer le mot de passe 2 fois (ou le champ vide le cas échéant) pour le confirmer).

Une fois générée, vous pouvez trouver cette clé dans votre dossier personnel/.ssh/id_rsa.pub : ouvrez ce fichier dans un éditeur de texte (ou si vous êtes sur Linux, faites cat ~/.ssh/id_rsa.pub dans le terminal) et copiez-en le contenu.

Pour lier la clé sur GitLab, il vous suffit de vous connecter, puis de cliquer sur votre photo de profil, puis sur Preferences. Allez ensuite dans SSH Keys (à gauche), puis entrez la clé précédemment copiée. Validez ensuite en cliquant sur Add key.

Quelques commandes de base

Récupérer un projet déjà existant

Rendez vous sur GitLab, puis choisissez un projet dans la liste, ouvrez le et cliquez sur Clone, puis Clone with SSH. Ouvrez ensuite un terminal et entrez la commande suivante : git pull {ici le lien que vous avez copié, sans les crochets}. Un nouveau dossier sera créé, comportant tous les fichiers de la repo Git.
